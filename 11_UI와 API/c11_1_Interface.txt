소프트웨어 공학에서의 인터페이스는 크게 UI와 API가 있다.

UI는 User Interface의 약자이며, 시스템과 일반 사용자 사이를 연결하는 입출력 매개에 해당한다.
API는 Application Programming Interface의 약자로, 시스템과 개발자 사이를 연결하는 매개에 해당한다.

예컨대 javascript:alert("Hello world"); 를 실행한다면 웹브라우저에서 출력되는 경고창은 일반 사용자가 사용하는 UI에 해당하고,
메소드 alert가 가지고 있는 기능(버튼, 경고창의 위치값, 아이콘 등: alert 내에 내장되어 있다)은 개발자가 사용하는 API에 해당한다.
이는 UI와 API가 계층적 구조를 가지고 있음을 뜻한다.

소프트웨어 공학을 도식화하면 다음과 같은 역피라미드형 구조가 된다.

사용자----------------------------
웹 개발자--------------------
브라우저-----------------
운영체제--------------
고급 언어----------
어셈블리어------
기계어-------

사용자와 하위 계층은 UI로 연결되고, 웹 개발자 이하의 하위 계층끼리는 API로 연결된다.
